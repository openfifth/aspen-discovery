### Account Updates
- Respect Allow Freezing Holds option in Library Settings. Applies to CARL.X, Polaris, Sierra, Symphony. (DIS-1634) (*MDN*)

### Administration Updates
- If an error occurs while submitting a form, fill the form with values entered by the user so they can correct and resubmit. (DIS-1589) (*MDN*)
  - This works both when creating a new object and when editing an existing object. 
  - Any errors that cause the object to not save are reported to the user. 
- Add proper return types and clean up syntax errors in Object Editor and derived classes. (DIS-1625) (*MDN*)
- Allow locking records in the Administration interface, and prevent users who do not have permissions from editing locked records. (DIS-1665) (*MDN*)
  - Records that can be locked are: Audiences, Basic Pages, Browse Categories, Browse Category Groups, Categories, Custom Forms, Custom Pages, Custom Web Resources Pages, Email Templates, Grapes Pages, Images, JavaScript Snippets, Layout Settings, PDFs, Placards, Quick Polls, System Messages, Themes, Web Resources, and Year In Review.
  - For users with permission to lock records, a Lock/Unlock button will display at the top of all edit pages. 
  - For users without permission, a message will be displayed at the top of all locked records stating that the record cannot be edited and all fields will be read only. 
  - In a list of all records, a new column will be added to indicate if the record is locked if one or more record in the list is locked.

### Indexing Updates
- Resolved facet-selection gaps where translated location names didn't match the facet label, so scoped availability now remains consistent after renaming branches. (DIS-1571) (*LS*)
  - Owning-location names now come directly from each Location's "Facet Label" or "Display Name" instead of the deprecated "location" Translation Map.
  - If no location code matches, likely due to a reliance on regex matching, fall back to using the "location" Translation Map.
  - If it is available, the Facet Label for the Location will be used for single-location Library Systems.

### Koha Updates
- Fixed Messaging Settings to honor the TalkingTech (itiva) phone transport and hide phone checkboxes on non-itiva-capable notices, matching Koha OPAC. (DIS-1643) (*LS*)
- Removed automatic insertion of default 9-5 library hours when no hours exist in the Location settings. (DIS-1604) (*LS*)

### Materials Requests Updates
- Fixed date filter in Manage Materials Requests to make the "To" date inclusive of the entire day, resolving issues where requests made on the current day were excluded from filter results. (DIS-1486) (*LS*)
- Fixed fatal error on Manage Requests page when displaying requests created by users without ILS barcodes. (DIS-1594) (*LS*)

### OverDrive Updates
- Fixed OverDrive availability counts showing double the actual number of copies and holds when libraries had access to both library-specific and shared collection records for the same title. (DIS-1606) (*LS*)

### Placards Updates
- Implemented scoring-based placard matching algorithm to ensure the most relevant placard is selected when multiple triggers could apply. (DIS-1644) (*LS*)
  - Relevance scoring now prioritizes match specificity, addressing previous issues where broad triggers (e.g., "lead") were incorrectly favored over more specific ones (e.g., "union leader").
  - Improved partial-match logic for scenarios such as search: "union lead" with triggers "union leader" and "lead," correctly prioritizing the more relevant "union leader."
  - Added debug information about the matched placard when "Show Debugging Information" is enabled in the IP Address settings.
- Fixed an issue where links inside a placard could break formatting. Placard links now apply only to the image if the body has any links. (DIS-1636) (*LS*)
- Text inside placards now follows the formatting you choose in the editor instead of always appearing bold. (DIS-1636) (*LS*)

### Polaris Updates

- Added Polaris User Defined Fields (UDFs) support for self-registration, allowing libraries to collect up to five custom fields from patrons. (DIS-1628) (*LS*)
