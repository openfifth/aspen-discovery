# Quarterly Backports

---

***The following notes include changes that were brought in early from 25.10.00. 
Please refer to the 25.09.00 and 25.09.01 release notes for more information about this release.***

## Aspen Discovery Updates

### Account Updates
- Correctly show recaptcha error message if recaptcha validation fails during self registration. (DIS-1384) (*MDN*)

### API Updates
- Fixed an issue where new translation terms from LiDA would not automatically populate in the Translations interface via translation API calls. (DIS-1400) (*LS*)
- Fixed an issue where browse categories made from advanced searches would display incorrect results via the API. (DIS-1242) (*LS*)
- Correctly generate bookcover links for suggested titles that have more than one format group within ListAPI. (DIS-1410) (*MDN*)

### Browse Categories Updates
- With "Use Accessible Layout" enabled for a Theme, browse categories on the home page now generate proper search URLs instead of placeholder links. (DIS-1301) (*LS*)

### Community Engagement Updates
- Fix issue loading campaigns if a patron has no patron type set. (DIS-1411) (*MDN*)
- Prevent upcoming campaigns with a start date of more than a month in the future from showing in the linked user section. (DIS-1425) (*AB*)
- Allow users to remove campaigns they are not interested in from the 'My Campaigns' section of their account. (DIS-1570) (*AB*)
- Allow admin to restore campaigns users have removed from the 'My Campaigns' section of their accounts. (DIS-1570) (*AB*)

### Digital Archives Updates
- Fixed Open Archives indexer to prevent image URLs (JPG, PNG, TIF, MP3, PDF) from being selected as record identifiers when they appear anywhere in the URL path. (DIS-1387) (*LS*)
- Fixed Open Archives cover images not displaying when URLs contain HTML entities. (DIS-1387) (*LS*)
- Added support for Omeka installation thumbnail extraction with new regex pattern for `<img class="full"...>` elements. (DIS-1387) (*LS*)

### Docker Updates
- Fix an error where the solr.xml was copied to the incorrect directory. (DIS-1414) (*LM*)
- Add a healthcheck for each solr core in the Solr Dockerfile. (DIS-1414) (*LM*)
- Parameterize host and port in the Solr Dockerfile. (DIS-1414) (*LM*)

### E-Commerce Updates
- Fixed an issue on the donations page where a currency code (e.g., "USD") was mistakenly passed as the locale, causing donations to fail, and updated it to use the proper locale format (e.g., "en_US"). (DIS-1337) (*LS*)
- Added null checks and proper error handling for Square payment processing on the donations page so that missing or incomplete fields no longer just trigger a generic AJAX error. (DIS-1337) (*LS*)
    - Updated Square order failure handling to return specific error messages instead of the vague "Method did not return results."
- Corrected a layout issue on the Donations creation page on smaller screens where address fields took up more vertical space than intended, blocking access to other fields. (DIS-1337) (*LS*)

### Grouped Work Display Updates
- Properly display MPAA Ratings for grouped works and MARC views. (DIS-1388) (*MDN*)

### Holds Updates
- Fixed an issue where patrons' home locations would be used for all item-level holds when selecting volumes, such as for Sierra libraries. (DIS-1327) (*LS*)
- Resolved an issue where non-holdable items would display as volume hold options for record variations. (DIS-1330) (*LS*)

### Koha Updates
- Fixed an issue where a password reset via Aspen to Koha would succeed, but an error message would display instead. (DIS-1383) (*LS*)
- Fixed an issue where password validation would be performed during self registration even though the field was not mandatory. (DIS-922) (*LS*)
- Properly load renewal information for checkouts including information about maximum renewals and auto-renewals. (DIS-17) (*MDN*)

### OverDrive Updates
- Remove OverDrive Holds Suspension Reactivation Date selection modal and related API parameters to align with OverDrive’s updated holds suspension behaviors. (DIS-1356) (*YL*)

### Palace Project Updates
- Implemented missing usage dashboard for Palace Project showing Active Users, Records With Usage, Checkouts, and Holds statistics across multiple time periods. (DIS-1376) (*LS*)

### Search Updates
- Correct display of the Collection facet. (DIS-1391) (*MDN*)
- Set display author as primary author to populate the SOLR 'author' fields, fixing the issue where Author search misses grouped works when the display author name differs from SOLR author fields. (DIS-1393) (*YL*)

### Sierra Updates
- Add deletion of bib records that have no MARC data returned from the API during the indexing.  (DIS-1296) (*YL*)

### Web Builder Updates
- Update the column types for grapes js to allow longer pages to be saved. (DIS-1458) (*AB*)

### Other Updates
- Update timestamp format when logging in Check Solr For Deleted Works Cron. (DIS-1389) (*MDN*)
- Add existence check for `$tmpLocation->code` when placing holds to prevent PHP warnings. (DIS-1327) (*LS*)
- Remove deprecated intval smarty modifier in duration.tpl. (DIS-1401) (*MDN*)
- Fixed physical description display on record pages to show complete MARC 300 field data by correcting field type comparison logic. (DIS-1354) (*LS*)
- Corrected import path in ILS/UsageGraphs.php that was preventing ILS usage analytics from loading. (DIS-1367) (*LS*)
- Fixed an issue where uploaded images, in particular the Logo within the Theme settings, were unproportionally enlarged. (DIS-1374) (*LS*)
- Fixed an issue where Themes settings would be cross contaminated during updates to CSS for all Themes. (DIS-1377) (*LS*)
- Only show Opens in New Window text for header menu links that open in a new window. (DIS-1420) (*MDN*)
- Make Hoopla status check consistent with retry interval. (DIS-1421) (*MDN*)
