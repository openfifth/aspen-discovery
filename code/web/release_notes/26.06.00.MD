#### Event Waiting Lists
##### Configuration
- If registration is enabled, administrators can enable a waiting list at the event level. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
- Administrators can decide on a set number of waiting list seats per event. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - The number of waiting list seats can be overridden for a specific date / time (a specific instance of an event).
- Administrators can configure how long an invitation to register from the waiting list remains valid (in hours). ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
##### User Experience
- Logged in patrons can join the waiting list of a full event. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - directly from the search results.
    - from an event's detail page.
    - or from their list of saved event instances.
- Patrons on a waiting list can see their position. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - directly from the search results.
    - from an event's detail page.
    - or from their list of saved event instances.
- Patrons can leave a waiting list. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - directly from the search results.
    - from an event's detail page.
    - or from their list of saved event instances.
- Waiting list seat availability is displayed. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - directly from the search results.
    - from an event's detail page.
    - or from their list of saved event instances.
- When a seat opens up (a registered patron unregisters, or an event's capacity is increased), the next patron on the waiting list is invited to complete their registration. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - the invited patron can register through the normal registration flow until their invitation expires.
    - if the invitation expires before the patron registers, the next patron on the waiting list is invited.
- This includes linked account support, which means: ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - patrons with linked users add and remove their linked users from the waiting list.
    - patrons with linked users can see whether their linked users are on a waiting list and, if so, in which position.
    - when a linked user cannot be reached for waiting list notifications, the primary account holder is contacted instead if they have a valid email and consented to email notifications for events.
#### Event Email Notifications
- Patrons can opt in to event email notifications from their account preferences. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
- Patrons who have opted in and have a contact email saved to their account receive an email when: ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
    - they are invited to register from the waiting list.
    - an event they are registered to is cancelled.
    - an instance of an event they are registered to is deleted.
- Patrons who have an open waiting list invitation see an actionable message on their MyAccount home page until they complete or decline their registration. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
- Patrons are informed when joining a waiting list whether their account is reachable for notifications, and are prompted to add an email and opt in if it is not. ([DIS-1909](https://aspen-discovery.atlassian.net/browse/DIS-1909)) (*CZ*)
#### Event Attendance Management 
##### Configuration
- Staff registration of patrons for events and event attendance management are now controlled by separate permissions, allowing libraries to grant either capability independently. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- The existing "staff registration only" event setting now only restricts patron self-registration; it no longer blocks staff from registering patrons. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
##### Staff Attendance Management UI
- Staff with the appropriate permission can access a new Attendance Management page from the admin area. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*JOM & CZ*)
    - the page lists upcoming registerable events that the staff member has location access to.
    - selecting an event shows the full registration list for that instance, with patron details (name, barcode, email) and registration metadata (when they registered, whether they attended).
- Staff can mark each registered patron as attended (or not) directly from the registration list. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*JOM & CZ*)
- Staff with the appropriate permission can register a patron for an event on their behalf. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*JOM & CZ*)
    - only events that are currently registerable are listed in the staff registration panel.
- Staff registration goes through the same flow as patron self-registration, so capacity and waiting list rules are respected (including waiting-list-aware capacity checks). ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- The registration list identifies which registrations were made by staff and which staff member performed them. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- Patrons registered by staff have the event added to their Saved Events list, flagged as having been saved by staff. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
    - staff are informed that a Saved Events entry will be created when they register a patron.
    - the entry is displayed in the patron's Saved Events as staff-saved.
- Cancelled registrations are no longer displayed in the patron's registration list. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
###### Event Registration List and CSV Downloads
- From the Attendance Management page for a specific event instance, staff can download the registration list for an event instance in either of two formats. ([DIS-1885](https://aspen-discovery.atlassian.net/browse/DIS-1885)) (*CZ*)
    - a plain-text list suitable for printing, with a checkbox per registration so attendance can be marked off by hand.
    - a CSV intended for spreadsheet use or further processing.
- Both downloads identify the event (title, date, time, library location) and include patron details (name, ILS barcode, email, post code, date of birth) and registration metadata (status, whether the registration was made by the patron or by a staff member — and which staff member — and the date registered). ([DIS-1885](https://aspen-discovery.atlassian.net/browse/DIS-1885)) (*CZ*)
    - the plain-text list shows the event details once in a header block above the rows; the CSV repeats them on every row so the file can be filtered or joined without losing context.
- The CSV additionally includes the event type, the recorded attendance state (Yes/No), the patron's submitted values for each custom registration field configured on the event type, and the attendee count submitted per attendee category assigned to the event type. ([DIS-1885](https://aspen-discovery.atlassian.net/browse/DIS-1885)) (*CZ*)

#### Custom Event Registration Fields
##### Configuration
- Each custom field is configured with an intended use: ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
    - "Event description section" — the existing staff-managed fields whose values are displayed publicly on the event page.
    - "Event registration form" — newly introduced patron-facing fields that appear on the registration form.
    - when building field sets, the field options available when building a field set are filtered to those whose use matches the set's use.
- Each event type is assigned one event information field set (retained existing behaviour) and one event registration field set (new field set type). ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
    - only field sets matching the relevant use are offered for each assignment.
##### Custom Registration Fields
- The fields from the registration field set assigned to an event's type are rendered on that event's registration form for patrons to fill in. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- Patrons' submitted values are persisted against their registration. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- Once a patron is registered, their submitted custom field values are displayed read-only on the registration view, both on the event page and from their list of saved event instances. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- The custom registration fields are only displayed when registration is currently available, or when the patron is already registered (so they can review their submitted values). ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
#### Multi-Attendee Event Registrations
##### Configuration
- Administrators can view/create/edit/delete event attendee categories (e.g. "Adults", "Children", "Carers") from a new admin page, accessible from the event administration section. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- Each event type can be assigned any number of attendee categories. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
    - per assignment, administrators can set a minimum and maximum count, and decide whether the category is required on the registration form.
    - each category can only be assigned once to a given type. Duplicates will not be saved.
##### User Experience
- When registering for an event whose type has attendee categories assigned, patrons enter a count for one or more category on the registration form. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
    - the category-specific min/max constraints configured for the event type are enforced on submission.
    - a tooltip clarifies that the patron should include themselves in the counts when relevant.
    - the system handles missing category information, treating the registration as a single seat.
    - events with no attendee category assigned are supported and also treat each registration as taking up a single seat.
- The category breakdown a patron submitted is displayed read-only on their registration view, both on the event page and from their list of saved event instances. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- Event capacity now reflects the total projected attendees across all registrations, not the number of registration records. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
    - available seats displayed on search results, event detail pages, and saved event instances all account for attendee counts.
    - registration is blocked when the requested attendee counts would exceed the remaining seats.
    - attendee categories are not used for waiting lists and do not impact waiting list behaviour. 
##### Staff Experience
- When a staff member registers a patron for an event, the same attendee category inputs are presented in the staff registration modal, with the same validation rules. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- The Attendance Management page displays the attendee category breakdown for the event instance (totals per category), as well as the per-registration breakdown for each patron. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
- When a registration is cancelled, its attendee count entries are removed so the event's projected attendance stays accurate. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
#### Staff Notes on Events
- Administrators can record internal notes directly on an event's edit form. The notes live on the event's settings page and are not surfaced anywhere else. ([DIS-1597](https://aspen-discovery.atlassian.net/browse/DIS-1597)) (*CZ*)
