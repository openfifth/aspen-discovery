## Aspen Discovery Updates
//mark n
### Analytics Updates
- If a library has Cookie Consent turned off, log interactions with eContent. ([DIS-1709](https://aspen-discovery.atlassian.net/browse/DIS-1709)) (*MDN*)

### Hold Updates
- Automatically load list of users to use when placing a hold if "Hide Pickup Location Prompt" is on. ([DIS-1724](https://aspen-discovery.atlassian.net/browse/DIS-1724)) (*MDN*) 

### Stripe Payment Updates
- Ensure that if a patron is blocked from fine payments (due to Sierra locks) the payment process stops. ([DIS-1623](https://aspen-discovery.atlassian.net/browse/DIS-1709)) (*LS*)

### Other Updates
- Visit Library option in the Locations Modal opens a new tab. ([DIS-1632](https://aspen-discovery.atlassian.net/browse/DIS-1632))
- Automatically insert translation terms in English when Google Translate is active when they are first viewed. ([DIS-1700](https://aspen-discovery.atlassian.net/browse/DIS-1700)) (*MDN*)
- When using the backupAspen script, export rows in chunks to not exceed max packet sizes on restore. ([DIS-1701](https://aspen-discovery.atlassian.net/browse/DIS-1701)) (*MDN*)

- Allow users to create events set in the past (DIS-1691) (*KL*)

### Stripe Payment Updates
- Stripe payment receipts are automatically captured and stored in Payment History and can be viewed after a successful payment. ([DIS-1623](https://aspen-discovery.atlassian.net/browse/DIS-1709)) (*LS*)
- Payment information in Stripe now includes a description and detailed metadata, such as transaction type, patron ILS ID, patron barcode, fine count, and fine IDs for better reporting. ([DIS-1623](https://aspen-discovery.atlassian.net/browse/DIS-1709)) (*LS*, *MDN*)
- Stripe payments now respect the system's configured currency code and symbol, not just USD. ([DIS-1623](https://aspen-discovery.atlassian.net/browse/DIS-1709)) (*LS*)
